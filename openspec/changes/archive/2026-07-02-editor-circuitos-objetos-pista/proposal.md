## Why

O editor de circuitos permite hoje criar apenas dois tipos de `ObjetoPista` (Escapada e Transparência), via um `if/else` fixo no botão "Criar Objeto". Quatro subclasses já existem e desenham corretamente (`ObjetoArquibancada`, `ObjetoConstrucao`, `ObjetoGuadRails`, `ObjetoPneus`), mas são código morto: não aparecem no combo de criação do editor, não têm exemplo em nenhum circuito salvo, e não são renderizadas durante a corrida — só `ObjetoEscapada` (debug) e `ObjetoTransparencia` (máscara de boxes) têm tratamento especial em `PainelCircuito`. Isso bloqueia a criação de cenário visual (arquibancadas, construções, guardrails, pilhas de pneus) usando os próprios objetos do jogo em vez de arte externa embutida na imagem de fundo, e cada novo tipo de objeto exigiria repetir esse trabalho manual de novo.

## What Changes

- Renomear `ObjetoGuadRails` para `ObjetoGuardRails` (corrige typo; não há circuitos salvos usando o nome atual, confirmado por busca nos XMLs de recursos, então não é necessário shim de compatibilidade).
- Substituir o `if/else` fixo de criação de objetos em `MainPanelEditor`/`FormularioObjetos` por um registro (factory/registry) de tipos de `ObjetoPista`, de forma que adicionar um novo tipo no futuro não exija tocar nesse fluxo de criação.
- Registrar `ObjetoArquibancada`, `ObjetoConstrucao`, `ObjetoGuardRails` e `ObjetoPneus` nesse registro, tornando-os selecionáveis e criáveis no editor de circuitos.
- Revisar as quatro classes para garantir consistência de comportamento com o padrão já usado por `ObjetoEscapada`/`ObjetoTransparencia` (redimensionamento, rotação, cor primária/secundária, área de seleção via `obterArea()`), preenchendo qualquer lacuna encontrada.
- Extrair a rotina de desenho procedural do circuito hoje presa em `MainPanelEditor` (traçado da pista/zebra/box e o loop que desenha todo `circuito.getObjetos()`, usada quando o editor está em modo "sem imagem de fundo") para um componente compartilhado, reutilizável fora do editor.
- Adicionar uma flag booleana em `Global` que, quando ativa, faz o carregamento da imagem de fundo do circuito (hoje um arquivo estático `circuitos/*_mro.jpg`) ser substituído pela geração em memória dessa mesma imagem usando o componente de desenho procedural acima — ou seja, a pista e os objetos de cenário (`ObjetoArquibancada`, `ObjetoConstrucao`, `ObjetoGuardRails`, `ObjetoPneus` e qualquer tipo futuro) aparecem no jogo exatamente como são vistos no editor, sem precisar de nenhum hook novo por tipo de objeto. `ObjetoEscapada` (marcador de debug) e `ObjetoTransparencia` (máscara funcional de boxes, já desenhada em tempo real por `PainelCircuito`) ficam explicitamente fora dessa geração, para não gravar permanentemente na imagem algo que já tem tratamento próprio.
- Aplicar essa flag tanto no carregamento local (`PainelCircuito`/`CarregadorRecursos.carregaBackGround`, modo solo e cliente Java do multiplayer) quanto no endpoint REST `/circuitoJpg/{nmCircuito}` (`LetsRace.java`), que serve a mesma imagem por HTTP para o cliente do multiplayer — gerando a imagem no servidor antes de responder, quando a flag estiver ativa.
- O carregamento da imagem real de referência dentro do próprio editor (usado quando o designer ativa "mostrar BG" para alinhar objetos sobre a arte original) não muda — continua lendo o arquivo do disco, já que sua finalidade é justamente comparar com a imagem original.
- Separar, no editor, os objetos de cenário (`ObjetoArquibancada`, `ObjetoConstrucao`, `ObjetoGuardRails`, `ObjetoPneus`) numa lista própria em `Circuito` (`objetosCenario`), do mesmo jeito que pista e box já são listas separadas — `ObjetoEscapada`/`ObjetoTransparencia` continuam na lista `objetos` existente. O editor passa a ter dois painéis de lista lado a lado (um para cada lista), em vez de um único painel combinado.
- Adicionar, no editor, um menu de contexto (clique direito sobre o objeto) para ajustar largura/altura/rotação diretamente, sem abrir o formulário modal completo.
- Adicionar, no editor, arrastar-e-soltar com o mouse (clicar e segurar sobre a área do objeto) para reposicionar um objeto, coexistindo com o atalho de teclado Alt+setas já existente.
- Adicionar, no editor (painel de controles, ao lado do checkbox "Noite"), duas caixas clicáveis de cor: uma para a cor de fundo de todo o circuito (`Circuito.corFundo`, já existente mas sem UI própria) e outra, nova, para a cor do asfalto da pista (`Circuito.corAsfalto`). Ambas usam `JColorChooser` e passam a valer tanto na pré-visualização do editor quanto na imagem gerada em memória (`DesenhoProceduralCircuito`).
- **BREAKING**: qualquer referência externa ao nome de classe `ObjetoGuadRails` (por exemplo em serialização XMLEncoder futura ou código de terceiros) deixa de resolver; nenhuma referência foi encontrada no repositório atual.

## Capabilities

### New Capabilities
- `objetos-cenario-circuito`: comportamento dos objetos de cenário de pista (`ObjetoArquibancada`, `ObjetoConstrucao`, `ObjetoGuardRails`, `ObjetoPneus`) — criação extensível no editor via registro de tipos, e aparição em corrida através da geração em memória da imagem de fundo do circuito (controlada por flag em `Global`), reproduzindo o desenho procedural já usado no editor.

### Modified Capabilities
(nenhuma — os requisitos existentes de `dev-editor-tools`, como diálogos de arquivo e localização de pacote, não mudam)

## Impact

- **Código afetado**: `src/main/java/br/f1mane/entidades/ObjetoGuadRails.java` (renomeada), `ObjetoArquibancada.java`, `ObjetoConstrucao.java`, `ObjetoPneus.java`; `src/main/java/br/f1mane/editor/MainPanelEditor.java`, `FormularioObjetos.java` (novo registro de tipos, e extração da rotina de desenho procedural para um componente compartilhado); `src/main/java/br/nnpe/Global.java` (nova flag); `src/main/java/br/f1mane/recursos/CarregadorRecursos.java` e `src/main/java/br/f1mane/visao/PainelCircuito.java` (geração em memória em vez de leitura de arquivo, quando a flag estiver ativa); `src/main/java/br/f1mane/servidor/rest/LetsRace.java` (endpoint `/circuitoJpg/{nmCircuito}` passa a gerar a imagem no servidor quando a flag estiver ativa); `Circuito.java` (novo campo `objetosCenario`); `FormularioListaObjetos.java` (passa a poder operar sobre qualquer uma das duas listas); `TipoObjetoPista.java` (nova flag indicando se o tipo é de cenário).
- **Persistência**: nenhuma mudança de schema necessária — a serialização via `XMLEncoder`/`XMLDecoder` é reflexiva; novos tipos são salvos/carregados automaticamente a partir do momento em que são instanciados pelo editor.
- **Circuitos existentes**: nenhum circuito salvo referencia os quatro tipos hoje, então não há migração de dados; os arquivos `circuitos/*_mro.jpg` continuam existindo e sendo usados normalmente quando a flag estiver desligada (comportamento padrão inalterado).
- **Modo solo/multiplayer**: `PainelCircuito` é usado no modo Swing solo (`MainFrame`) e pelo cliente Java do multiplayer (`AppletPaddock`/`JogoCliente`); a geração em memória, quando ativa, passa a valer nos dois, e também no endpoint REST que serve a imagem para o cliente do multiplayer. O cliente web HTML5 (`html5/*.html`, renderização em canvas/JS) não usa `PainelCircuito` e fica fora do escopo desta mudança.
